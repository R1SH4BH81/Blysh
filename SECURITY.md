# Security Policy

## Supported Versions

We actively provide security updates for the current major version. Since **Blysh** maintains parity between Python and JavaScript packages, the support cycle applies to both.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.** If you discover a potential security flaw in this project, please help us protect the community by following these steps:

### 1. How to Report
You can report vulnerabilities privately via:
* **GitHub Private Reporting:** Navigate to the [Security tab](https://github.com/R1SH4BH81/Blysh/security/advisories/new) of this repository and select "Report a vulnerability".
* **Direct Contact:** Reach out to the maintainer, [RISHABH MISHRA](https://github.com/R1SH4BH81), directly through GitHub profile contact methods.

### 2. What to Include
To help us resolve the issue quickly, please include:
* A brief description of the vulnerability.
* Steps to reproduce the issue (PoC code or clear instructions).
* Potential impact (e.g., remote code execution, XSS, etc.).

### 3. Our Response Process
* **Acknowledgement:** You will receive an initial response within **48–72 hours**.
* **Fix & Timeline:** We aim to provide a patch or mitigation strategy as quickly as possible. We will keep you updated throughout the process.
* **Disclosure:** Once the vulnerability is fixed, we will coordinate a public disclosure and credit you for your contribution (unless you prefer to remain anonymous).
